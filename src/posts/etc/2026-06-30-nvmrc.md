## .nvmrc이란

`.nvmrc`는 프로젝트에서 사용할 Node.js 버전을 명시해두는 파일.

Node 버전 관리 도구들이 이 파일을 읽어서 자동으로 맞는 버전을 잡아 주는 역할을 한다.

팀 프로젝트나 오픈소스에서 가장 자주 일어나는 문제 중 하나가 Node 버전 차이인데, `.nvmrc`가 그걸 방지해준다.

## 사용법

프로젝트 루트에 `.nvmrc` 파일을 만들고 버전만 적으면 끝이다.

```
20.11.0
```

## 현재 버전으로 .nvmrc 만들기

지금 쓰고 있는 버전을 그대로 파일로 남기고 싶을 때:

```bash
node -v > .nvmrc
```

`node -v`는 `v20.11.0` 형태로 출력하는데, 앞에 `v`가 있어도 대부분의 도구가 잘 읽는다.

## `.nvmrc` vs `engines` 필드

`package.json`에 `engines`를 쓰는 방법도 있다.

```json
{
  "engines": {
    "node": ">=20.0.0"
  }
}
```

차이는 이렇다:

- `engines`: 지원하는 버전 범위를 명시. `npm install` 시 경고를 줄 수 있지만 강제로 전환하진 않는다.
- `.nvmrc`: 정확한 버전을 고정하고 버전 관리 도구가 자동으로 맞춰준다.

둘 다 같이 쓰는 경우도 많다. `.nvmrc`로 개발 환경 버전을 고정하고, `engines`로 지원 범위를 문서화하는 식으로.

## 정리

- `.nvmrc` = 프로젝트 루트에 Node 버전 하나 적어두는 파일
- 팀 프로젝트에서 버전 충돌 방지에 유용
- `engines` 필드와 함께 쓰면 더 명확하게 버전 의도를 남길 수 있다
